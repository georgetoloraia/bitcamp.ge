#### პროექტი 126 - Real-time Chat Application

შექმენით ჩატის (სასაუბროს) მაგვარი აპლიკაცია. მომხმარებლებს უნდა შეეძლოთ რეგისტრაცია, დალოგინება და შემდეგ სხვადასხვა სასაუბრო ოთახებში შესვლა და შეტყობინებების გაგზავნა.
ეს პროექტი მოიცავს Django Channels-ის გამოყენებას websocket-ებთან ერთად, ხოლო მომხმარებლების მენეჯმენტისთვის და ოთახების API-ისთვის Django Rest Framework-ს აწ უკვე ნაცნობ ფუნქციონალს.

ფუნქციონალი:
- Real-time შეტყობინებები სხვადასხვა სასაუბრო ოთახებში (იგივე ჯგუფებში, group chat).
- რეგისტრაცია და ავტორიზაცია DRF მეშვეობით.

ბონუს ჩელენჯები:
- შექმენით მარტივი ფრონტენდი თქვენი აპლიკაციისთვის.
- შექმენით ფაილების და ფოტოების გაგზავნის ფუნქციონალი.


#### პროექტი 127 - E-commerce Platform API

შექმენით სრულყოფილი E-commerce ვებსაიტის API.
ფუნქციონალი:
- რეგისტრაცია და ავტორიზაცია.
- პროდუქტებზე ინფორმაციის წამოღება.
- "კალათაში დამატება" და ამ კალათის მენეჯმენტის ფუნქციონალი.
- შეკვეთების დამუშავება.
- მომხმარებლების პროფილი.
- პროდუქტების მოძებნა გასაღები სიტყვებით ან კატეგორიებით.

ბაზის არქიტექტურა:
- შეზღუდულები არ ხართ მოდელების სქემის მოფიქრებაში. ეცადეთ რაც შეიძლება გამართული, ლოგიკური და ჩამოყალიბებული მოდელების სტრუქტურა მოიფიქროთ.

პროექტის დასრულების და გატესტვის შემდეგ, აუცილებლად უნდა მოხდეს მისი კონტეინერიზაცია Docker-ის გამოყენებით.
დასრულებული პროექტი ატვირთეთ github-ის რეპოზიტორიაში და აღწერეთ შესაბამისი README ფაილი.
არ დაგავიწყდეთ environment variables-ის გამოყენება.
აუცილებლად გამართეთ თქვენი პროექტის CI github actions-ის გამოყენებით.
